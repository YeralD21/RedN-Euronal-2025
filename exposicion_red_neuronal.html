🧠 Red Neuronal para Optimización de Servicios en la Nube

Sistema de Inteligencia Artificial para Colocación Óptima de Servicios

¿Qué es este proyecto?

Este proyecto implementa una red neuronal artificial que utiliza técnicas de machine learning para determinar automáticamente si la ubicación actual de un servicio en la nube es óptima o no, basándose en múltiples métricas de rendimiento del sistema.

🎯 Objetivo Principal

Optimizar automáticamente la colocación de servicios en entornos de computación en la nube y edge computing

🤖 Tecnología

Red neuronal multicapa implementada en PyTorch con funciones de activación ReLU y Sigmoid

📊 Aplicación

Análisis en tiempo real de métricas de rendimiento para toma de decisiones automática

Ajuste del Umbral de Decisión

Impacto del Umbral en las Métricas

🔻 Umbral Bajo (0.3)

Ventajas:
  • Detecta más servicios como óptimos
  • Reduce falsos negativos
  • Mayor recall (sensibilidad)
Desventajas:
  • Aumenta falsos positivos
  • Menor precisión
  • Riesgo de mantener servicios problemáticos

⚖️ Umbral Moderado (0.5)

Ventajas:
  • Balance entre precisión y recall
  • Decisiones equilibradas
  • Estándar en clasificación binaria
Consideraciones:
  • Puede no ser óptimo para todos los casos
  • Requiere evaluación específica del dominio

🔺 Umbral Alto (0.8)

Ventajas:
  • Mayor precisión en positivos
  • Reduce falsos positivos
  • Decisiones más conservadoras
Desventajas:
  • Aumenta falsos negativos
  • Menor recall
  • Puede reubicar servicios óptimos

¿Cuándo Ajustar el Umbral?

🛡️ Escenario Conservador

Umbral Alto (0.7-0.8)

Usar cuando los falsos positivos son costosos:

  • Aplicaciones críticas (salud, finanzas)
  • Costos altos de mantener servicios problemáticos
  • Preferir reubicar servicios dudosos

⚖️ Escenario Equilibrado

Umbral Moderado (0.5)

Usar cuando se busca balance general:

  • Aplicaciones web estándar
  • Costos similares para ambos tipos de error
  • Decisiones automáticas rutinarias

🚀 Escenario Agresivo

Umbral Bajo (0.3-0.4)

Usar cuando los falsos negativos son costosos:

  • Servicios con alta disponibilidad requerida
  • Costos altos de reubicación
  • Preferir mantener servicios funcionando

Implementación en el Código

Código Actual (Umbral fijo 0.5):

def predecir_servicio(valores):
    prediccion = modelo(datos_tensor)
    return int(prediccion.item() > 0.5)  # Umbral fijo

Código Mejorado (Umbral ajustable):

def predecir_servicio(valores, umbral=0.5):
    prediccion = modelo(datos_tensor)
    probabilidad = prediccion.item()
    return int(probabilidad > umbral), probabilidad

Matriz de Confusión

La matriz de confusión es una herramienta fundamental para evaluar el rendimiento de nuestro modelo de clasificación. Nos permite visualizar cómo se comporta el modelo con cada clase y identificar qué tipos de errores comete.

¿Qué es una Matriz de Confusión?

Es una tabla que muestra las predicciones correctas e incorrectas del modelo comparadas con las etiquetas reales. En nuestro caso, como es clasificación binaria, tenemos una matriz de 2x2.

🔍 Definición de Cada Cuadrante:

🎯 Verdaderos Positivos (TP)

Definición: Servicios que realmente SON óptimos y el modelo los predice como óptimos.

En tu proyecto: Servicios con buenas métricas que el modelo correctamente identifica como "mantener en ubicación actual".

Resultado: ✅ Predicción CORRECTA

✅ Verdaderos Negativos (TN)

Definición: Servicios que realmente NO son óptimos y el modelo los predice como no óptimos.

En tu proyecto: Servicios con métricas problemáticas que el modelo correctamente identifica como "necesitan reubicación".

Resultado: ✅ Predicción CORRECTA

⚠️ Falsos Positivos (FP)

Definición: Servicios que realmente NO son óptimos pero el modelo los predice como óptimos.

En tu proyecto: Servicios problemáticos que el modelo incorrectamente dice "mantener en ubicación actual".

Resultado: ❌ Predicción INCORRECTA - Error Tipo I

❌ Falsos Negativos (FN)

Definición: Servicios que realmente SÍ son óptimos pero el modelo los predice como no óptimos.

En tu proyecto: Servicios funcionando bien que el modelo incorrectamente dice "necesitan reubicación".

Resultado: ❌ Predicción INCORRECTA - Error Tipo II

Cálculo de Métricas desde la Matriz:

Accuracy (Precisión General)

¿Qué mide? Porcentaje total de predicciones correctas (tanto óptimas como no óptimas).
Accuracy = (TP + TN) / (TP + TN + FP + FN)
Accuracy = (82 + 89) / (82 + 89 + 12 + 18) = 171/201 = 85%

Precision (Precisión de Positivos)

¿Qué mide? De todos los servicios que el modelo dice que son óptimos, ¿cuántos realmente lo son?
Precision = TP / (TP + FP)
Precision = 82 / (82 + 12) = 82/94 = 87%

Recall (Sensibilidad)

¿Qué mide? De todos los servicios que realmente son óptimos, ¿cuántos logra identificar el modelo?
Recall = TP / (TP + FN)
Recall = 82 / (82 + 18) = 82/100 = 82%

Specificity (Especificidad)

¿Qué mide? De todos los servicios que realmente NO son óptimos, ¿cuántos logra identificar correctamente el modelo?
Specificity = TN / (TN + FP)
Specificity = 89 / (89 + 12) = 89/101 = 88%
